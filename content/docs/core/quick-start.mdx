OpenHive is a framework designed to enable seamless communication and collaboration between distributed AI agents.
It provides a standardized framework for agent identity, message formats, and capability discovery, allowing agents
built by different developers on different platforms to interoperate reliably.

## Features

<Cards>
  <Card icon={<Cpu className="text-green-500" />} title="Agent Core">
    Handles most of the logic, agent search, communication, task processing, capability decalaration.
  </Card>

<Card icon={<Globe className="text-purple-500" />} title="Registry">
  Central entity for offline/online agent regsitration, agent discovery and
  search by capabilities.
</Card>

<Card icon={<Infinity className="text-cyan-500" />} title="Agent Server">
  Creates an internal server, allowing interaction between your agent and others
  in your cluster.
</Card>

<Card icon={<Shield className="text-orange-500" />} title="Agent Signature">
  Communication between agents is automatically signed by agent's keys to ensure
  authenticity and trust.
</Card>

<Card
  icon={<Waypoints className="text-green-500" />}
  title="Framework Agnostic"
>
  Supports most popular frameworks to integrate with, in both (Node.js and
  Python).
</Card>

  <Card icon={<Terminal className="text-blue-500" />} title="OpenHive CLI">
    A command line tool to run local registry, create & build agents, useful for testing and deployment.
  </Card>
</Cards>

## Create your first agent

<Steps>
  <Step>
    ### Setup the CLI
    
    ```bash
    npm i -g @open-hive/cli
    ```

    This will make the `hive` tool available in your terminal.
    ```bash
    hive --version
    @open-hive/cli/0.4.0
    ```

  </Step>
  <Step>
    ### Create agent

    <Tabs items={['Node', 'Python']}>
      <Tab value="Node">
        ```bash
        hive create agent first-agent --node
        ```

        ![Create Node Agent](/hive-create-agent-node.gif)
      </Tab>
      <Tab value="Python">
        ```bash
        hive create agent first-agent --python
        ```

        ![Create Python Agent](/hive-create-agent-python.gif)
      </Tab>
    </Tabs>

  </Step>
  <Step>
    ### Review your agent structure

    <Tabs items={['Node', 'Python']}>
      <Tab value="Node">
        <Files>
          <Folder name="src" defaultOpen>
            <File name="index.ts" />
          </Folder>
          <File name=".env" />
          <File name=".hive.yml" />
          <File name="package.json" />
          <File name="tsconfig.json" />
        </Files>
      </Tab>
      <Tab value="Python">
        <Files>
          <File name=".env" />
          <File name=".hive.yml" />
          <File name="main.py" />
          <File name="requirements.txt" />
        </Files>
      </Tab>
    </Tabs>

  </Step>
  <Step>
    ### Run the local registry

    The local registry is a special agent that runs on your machine and is used for agent discovery.

    ```bash
    hive start
    ```

    ![Start Local Registry](/hive-start.gif)

    This will start a local server on `http://localhost:11000`.

  </Step>
  <Step>
    ### Run your agent

    Now that you have a registry running, you can start your agent.

    <Tabs items={['Node', 'Python']}>
      <Tab value="Node">
        ```bash
        cd first-agent
        npm install
        npm run dev
        ```
      </Tab>
      <Tab value="Python">
        ```bash
        cd first-agent
        pip install -r requirements.txt
        python main.py
        ```
      </Tab>
    </Tabs>

    Your agent will start and register itself with the local registry.

  </Step>
  <Step>
    ### Next steps

    You can now:
    - Inspect your agent's capabilities in `src/index.ts` or `main.py`.
    - Create a second agent and make them communicate.
    - Explore the documentation for more advanced features.

  </Step>
</Steps>
