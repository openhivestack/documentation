The H.I.V.E. Protocol defines how AI agents can work together to get tasks done efficiently. This specification provides a streamlined approach designed for scale and rapid adoption.

## Design Philosophy

H.I.V.E. prioritizes **simplicity over features** to maximize adoption:

- **Efficient Messaging**: Compact message format for reduced bandwidth usage
- **Direct Task Processing**: Simple task agreements instead of formal contracts
- **Three Implementation Tiers**: Choose your complexity level (Minimal, Standard, Full)
- **HTTP-First**: Uses familiar web technologies developers already know

## Overview

The H.I.V.E. Protocol uses a streamlined architecture with two main components:

- **Communication Layer**: Handles how agents connect and identify each other
- **Agent Collaboration**: Manages task negotiation, execution, and completion

This streamlined design makes it easy to implement while ensuring agents can work together effectively at scale.

## 1. Communication Layer

Agents communicate using standard web technologies like HTTP and WebSockets. The focus is on simplicity and
compatibility with existing infrastructure.

## 2. Agent Identity & Discovery

### 2.1. Agent Identification

- Agents identify themselves using Ed25519 key pairs (recommended) or RSA
- Agent ID format: `hive:agentid:uniqueIdentifier` (e.g., `hive:agentid:abc123`)
- Messages are signed with the private key for verification
- No complex DID resolution - simple and fast

### 2.2. Discovery

- **Primary**: HTTP registry service for capability lookup
- **Fallback**: Direct endpoint connections for known agents
- **Optional**: mDNS for local network discovery
- Registry uses simple REST API: `GET /agents?capability=text-translation`

### 2.3. Capability Sharing

Agents describe capabilities in simple format:

```json
{
  "id": "text-translation",
  "input": { "text": "string", "target_lang": "string" },
  "output": { "translated_text": "string" }
}
```

- Fast discovery and matching
- Easy to generate programmatically

## 3. Task Collaboration

### 3.1. Simple Message Format

All messages use this minimal format:

```json
{
  "from": "hive:agentid:abc123",
  "to": "hive:agentid:xyz789",
  "type": "task_request",
  "data": {
    "capability": "text-translation",
    "text": "hello",
    "target_lang": "es"
  },
  "sig": "ed25519_signature"
}
```

### 3.2. Lightweight Task Agreements

Use simple task agreements:

```json
{
  "task_id": "task-123",
  "capability": "text-translation",
  "params": { "text": "hello", "target_lang": "es" },
  "deadline": "2025-07-18T15:00:00Z"
}
```

**Benefits**:

- No contract negotiation overhead
- Faster task initiation
- Simpler implementation

### 3.5. Security

- **Required**: TLS encryption for all communications
